◆ get_frame_lower_boundary()

static unsigned get_frame_lower_boundary ( unsigned  initial_frames_count,
unsigned  element_index 
)
inlinestatic

Returns first frame of sequence of frames involving the work with element element_index of (TrashGuyState::text).
You can perceive this function as a parabola, where each x is element_index and y is returned frame:
frame = element_index2 + (initial_frames_count - 1)element_index
See this visualization.
For example:
(get_frame_lower_boundary(const, i + 1) - get_frame_lower_boundary(const, i))
will yield number of frames needed to process element with index i.
Initial number of frames is computed as:
(spacing + 1) * 2

element_indexindex of text element in TrashGuyState::text[element_index]
Returns
first frame of processed element_index

◆ tguy_set_frame()

void tguy_set_frame ( TrashGuyState *  st,
unsigned  frame 
)

In order to properly set frame we need to know few things beforehand:

  1. element_index for TrashGuyState::text[element_index] we're currently working on
  2. number of frames per element
  3. first frame (boundary) involving the work on element with element_index
  4. sub frame (frame index)
  5. direction
  6. index within the field

All we know is desired frame and number of frames per first element.

  1. Because of the get_frame_lower_boundary() we know that initial frame of each element is computed as:
    frame = element_index * (initial_frames_count + element_index - 1)
    which is equivalent to this quadratic equation:
    (element_index)2 + (initial_frames_count - 1)element_index - frame = 0
    We solve this equation for element_index, which is x1.
    (x1 means right wing of the parabola, x2 (left side) is meaningless to us, valid indices/frames only reside on the right side)
    x1 = (-b + sqrt(b2 - 4ac)) / 2a
    but because c is always negative and a is always 1, we can rewrite it as:
    x1 = (sqrt(b2 + 4c) - b) / 2 , which is the final formula
  2. Simple arithmetic progression, with each next element number of frames increases by 2.
  3. We now have all the values needed to call get_frame_lower_boundary().
  4. boundary <= (boundary + sub_frame) < boundary + frames_per_element ,
    thus sub_frame = frame - boundary
  5. we spend the same number of frames moving left/right, if sub_frame < (total / 2) -> right, else -> left
  6. index i within the field is computed as sub_frame % (total / 2)
